A. 快

题目描述

​ 蔡老板有$n$个豪宅，第$i$个坐标为$(a_i,b_i)$，每个豪宅有一个加油站，单价为$c_i$，加油站里面有无限的油。

​ 第$i$个豪宅到到第$j$个豪宅的距离$d(i,j)$为$|a_i-a_j|+|b_i-b_j|$。蔡老板想开豪车从第$1$个豪宅到第$2$个豪宅，豪车的油箱容量为$W$，每开$1$的单位距离需要消耗$1$单位的油，在行驶过程中需要保持油箱的油量非负。由于油箱容量限制，所以蔡老板可能不能直接从第$1$个豪宅开到第$2$个豪宅，需要在中间找一些豪宅作为停靠点加油。在第$i$个豪宅加$k$的油需要的代价为$c_i\times k$。

​ 一开始油箱的容量是空的，也就是需要在第$1$个豪宅加油。蔡老板想知道不加超过$d$次油，也就是经过不超过$d−1$个中间停靠点，从起点到终点所需要的最小总代价。

输入格式

第一行一个整数$n$。

接下来$n$行每行三个整数$a_i,b_i,c_i$。

接下来一行两个整数$W,d$。

输出格式

一个整数，表示答案。如果无解，输出$−1$。

样例输入

样例输出

数据范围与限制

对于所有数据，$1\leq n \leq 1000,1\leq W \leq 10^5,0\leq a_i,b_i \leq 10^6 ,1\leq c_i \leq 10^4,1\leq d \leq 10$。

子任务1(10分)：$n\leq 10$。

子任务2(20分)：$n,W\leq 100$。

子任务3(30分)：$n\leq 100$。

子任务4(40分)：无额外限制。

时间限制：1s

空间限制：1GB

sol：

注意到在一个点，要么加满或者加油到恰好能到下一个点，这是因为如果固定了路径，并且路径里面有可到达的费用比当前小的点，那么就会加油到那个点，否则就会直接加满。

有了上述观察之后，考虑$dp_{i,j}$表示上一个停靠点是$i$，加满油到达$j$。以及$dp2_i$表示到达$i$时油箱恰好为$0$。

考虑这些东西之间的转移，枚举下一个停靠点$k$，然后考虑分别以哪种状态（加满或者刚好放空）转移过去。

共四种转移：

如$i,j$表示$i$加满到达$j$，$j,k$ 表示$j$加满到达$k$。$i,j$和$j$转移到$j,k$就只要算在$j$处加满的最小代价。$j$转移向$k$，两重循环即可。较困难的是从$(i,j)$（加满过来）转移到$k$（刚好放空），这个时候需要加的油是$(dis(j,k)-(W-dis(i,j)))\cdot c_j$，总代价还要加上一个$dp_{i,j}$。然后注意到转移时要满足$dis(j,k)\ge W-dis(i,j)$。然后前面那个式子可以拆成$dis(j,k)\cdot c_j$，和$-(W-dis(i,j))\cdot c_j +dp_{i,j}$，这一部分是与$i$相关的。那么就相当于每次查询满足$dis(j,k)\ge W-dis(i,j)$的$i$的最小值。这个可以把所有的状态排序之后每次查询前缀最小值。

时间复杂度：$O(n^2 \log n+n^2 d)$。

B.乐

题面描述：

​ 你有一个$n$个点带标号的无向完全图。对每一条边，你想给它赋一个$1\sim k$的正整数权值。

​ 想知道有多少赋值方法使得整个图有唯一的最小正整数。输出答案模$P$的值。

输入格式

​ 第一行三个整数，$n,k,P$，保证$P$为素数。

输出格式

​ 一行一个整数，表示答案。

样例输出

样例输出

数据范围与限制

​ $10%,n\leq 5,k\leq 5$。

​ $20%,n\leq 5$。

​ 另外$10%,n\leq 10,k\leq 10$。

​ 另外$10% ,n\leq 10$。

​ 另外$10%, n\leq 20,k\leq 100$。

​ 另外$10%,n\leq 20$。

​ 另外$20%,k\leq 100$。

​ $100%,2\leq n \leq 50,1\leq k \leq 10^9 ,\max(n,k)+1 \leq P\leq 10^9$。

​ 时间限制：2s

​ 空间限制：1GB

sol：

首先可以想象，其实确定了这些边的大小关系，就能知道生成树是否唯一，假设第一小的边是哪些集合，第二小的边是哪些集合……枚举完之后即可知道生成树是否唯一。

所以一个自然推论就是考虑$F(n,m)$表示$n$个点每条边边权在$1\sim k$之间，最小生成树上的边都不超过$m$，最小生成树唯一的方案数。

先考虑权值较小时的情况：

考虑边权恰好为$m$的边，删除之后，假设树连通块大小为$x_1,x_2,\ldots x_r,\sum\limits_{i=1}^r x_i= n$。先看分点集的方案数是$\binom {n-1}{x_1-1} \binom {n-x_1-1} {x_2-1} \cdots $（至于为什么要减1是因为比如$x_1=2,x_2=2$，那么$(1,2),(3,4)$就会被算两次….不太懂….）。然后由于海莱定理（？，不同连通块之间的树边的连法一共有$\left(\prod x_i \right)\left(\sum x_i =n \right) ^{r-2}$种。不同连通块之间的非树边一共有$u=\binom n 2 -\sum \binom {x_i} {2}-r+1$条，那么方案数为$(k-m)^u$（非树边一定$\ge m$），然后这里$x_i$的贡献为

$(k-m)^{-\sum \binom {x_i}{2}}$

。每个连通块内部的连法为$F(x_i,m-1)$，总共就是$\prod F(x_i,m-1)$。所以使用背包计算上述式子即可。

然后大胆猜想这是一个这是一个关于$m$的多项式，每个项的方案对答案的贡献$F(n,*)$是一个次数不超过$\binom n 2$的多项式（因为总共就$\binom n 2$条边）。

于是考虑$dp$求出小的$m$的答案然后插值即可。

时间复杂度$O(n^5)$。

C.题

题目描述：

​ 你有两个字符串$S,T$，下标从$1$开始标号。

​ 有$q$组询问，每次询问给$a,b,c,d$，问$S[a\ldots b]$和$T[c\ldots d]$的最长公共子序列长度。

输入格式

​ 前两行，每行一个小写字符串$S,T$。

​ 接下来一行一个整数$q$。

​ 接下来$q$行，每行四个整数$a,b,c,d(1\leq a\leq b \leq |S|,1\leq c\leq d\leq |T|)$。

输出格式

​ $q$行，每行一个数表示答案。

样例输入

样例输出

数据范围

​ $10%$，$|S|,|T|,q\leq 500$。

​ 另外$30%,q\leq 500$

​ 另外$30%$，$|S|,|T|\leq 500$。

​ $100%,1\leq |S|,|T|\leq 3000,1\leq q \leq 5000$

​ 时间限制：3s

​ 空间限制：1GB

sol

先考虑最初始的$dp$为$dp_{i,j}=\max{dp_{i-1,j},dp_{i,j-1},dp_{i-1,j-1}+[S_i= T_j] }$

考虑$dp$转移的图，是一个网格图。里面大概这样连边：

等价于要求网格图中$(a,c)\to (b,d)$的最长路。

考虑分治（类似于ZJOI2016旅行者，考虑一排中间点，对于每一个中间点求出左边点到他的最长路，对于右边点也求出最长路，查询的时候就枚举怎么通过分界线的（两边加起来取$\max$）。如果直接做，那么复杂度是$O(n^3+qn)$，我们需要优化前一部分的复杂度。

对于每个跨过中线的问题，考虑其中的一部分，我们要求$(x,y)$到$(m,)$到$(x,y)$的最长路，等价于求$s[m\ldots x]$和$t[ \ldots y]$的LCS。

考虑从$t$开头删字符，维护和$s$所有前缀的LCS。

考虑从$t$开头删除一个字符，会减小的部分一定如下图所示单调区域。（一个稍微好理解的方法是这个图中的最长路径树不会相交）所以可以每次通过往右往下走的方式在线性时间内找到这个区域。然后通过维护每一项与上面与左边的差值的方法，线性地完成修改。

接下来考虑如何查询，因为我们没有显式的维护出$dp$数组，但是只要记录一下每个时刻每行减小了的那个前缀。最后查询的时候扫一下就能知道每个时刻有没有减小。这样前面的问题时间复杂度被优化到了$O(n^2 \log n+ qn)$。

A.划分

题面描述

​ 你有一棵$n$个点的树，每个点有个点权。

​ 把它划分成$k$个连通块，每个连通块的权值为这里面所有点的权值和的平方。一个划分方案的权值为所有连通块的权值和。

​ 对$k=1,2,\ldots ,n$，求出将一棵树划分为$k$个连通块的最小代价。

​ 数据保证随机，具体见输入格式部分

输入格式

​ 第一行一个正整数$T$，表示数据组数。

​ 对于每组数据，第一行一个正整数$n$。

​ 接下来一行$n$个整数，表示$1,2,\ldots ,n$号点的点权。

​ 接下来$n-1$行，每行两个整数，表示一条树边。

​ 保证点权在$[1,10^6]$之间随机，保证树在所有有标号的无根树里面均匀随机（也就是随机一个Prufer序列，然后生成对应的无根树）。

输出格式

​ 对于每组数据，输出一行$n$个整数，表示$k=1,2,\ldots ,n$时候的答案。

样例输入

样例输出

数据范围

​ $10%,n\leq 20$

​ $30%,n\leq 50$

​ $60%,n\leq 100$

​ $80%,n\leq 200$

​ $100%,2\leq n\leq 300,T\leq 10$

​ 时间限制：2s

​ 空间限制：1GB

sol：

考虑$dp_{i,j,k}$表示$i$这个子树，切了$j$段，当前连通块大小为$k$的最小和。

容易发现，这个状态过多，所以不行。将$(k,dp_{i,j,k})$记作二元组，考虑要保留哪些二元组。

容易发现，每个连通块，一个子树合并进去，相当于$(x,y)$和另一个点对$(u,v)$合并，权值和为$(x+u)^2+y+v$，也就是$x^2+2xu+u^2+y+v$。这其实是一个斜率优化的形式。当$u,v$为定值的时候，相当于$(x^2+y,x)$点（？乘上$(1,2u)$的最小值，这个东西的最小值一定在下凸壳上，所以只需要保留$(x^2+y,x)$（？下凸壳上的递减段部分即可。对每个$(i,j)$保留它在下凸壳上的一个状态即可$k_1^2+dp_{i,j,k_1}$。

B.随机游走

题面描述：

​ 你有一棵$n$个节点的有根树，$1$号点为根节点。你要在上面随机游走，每次你到一个点，会等概率随机走向其中的一个邻居(包括它的父亲)。

​ 记$E(s,t)$表示从$s$出发随机游走，第一次到达$t$的步数平方的期望。

​ 记$sub(x)$表示$x$这个子树中的所有节点（包括$x$）。

​ 现在有$q$组询问，每次给你$u,v$，保证$sub(u)$和$sub(v)$没有交集，求$\sum\limits_{x\in sub(u),y\in sub(v)} E(x,y)$。

​ 由于答案可能很大，对$998244353$取模。如果是分数$\frac{x}{y}$，输出$x$乘$y$的逆元。

输入格式

​ 第一行两个整数$n,q$。

​ 接下来$n-1$行，每行两个整数，表示一条树边。

​ 接下来$q$行，每行两个整数，表示一个询问。

输出格式

​ 对于每个询问，输出一个数表示答案。

样例输入

样例输出

数据范围

​ $20%,n,q\leq 10$

​ 另外$20%,n\leq 100,q\leq 1$

​ 另外$30%,n\leq 10^5,q\leq 1$

​ $100%,2\leq n\leq 10^5,1\leq q\leq 10^5$。

​ 时间限制：2s

​ 空间限制：1GB

sol：

对于一次方比较好做，只要求出每个点到它的父亲期望要走多少步，然后把路径上所有边的期望加起来就是。

二次方可能就会麻烦一点，可能会有别的推法，这里讲一个可能通用一点的方法。

考虑计算每条边从$u$走到$v$的概率生成函数。记$p_i$表示从$u$走到$v$走$i$步，$P(x)=\sum p_i \cdot x^i$。

然后这个东西有个常用的性质$P(1)=1$（所有概率加起来一定等于$1$）。

而$P’(1)=\sum i\cdot p_i\cdot x^{i-1}=\sum i \cdot p_i$这个东西就表示$x$的期望（$E[x]$）。

那么二阶导$P’’(1)=\sum i\cdot (i-1)\cdot p_i\cdot x^{i-2}=\sum i\cdot (i-1) \cdot p_i$

平方就相当于$P’(1)+P’’(1)$

考虑如何计算往上走一步（走到父亲），记$d$为$u$点的度数，那么它的概率生成函数有$f_u=\frac{1}{d}\left(x+\sum\limits_{v} (f_v \cdot x)\cdot f_u \right)$（走到每一个点的概率都是$\frac{1}{d}$，第一个$x$表示一步就走到了，否则就是走到一个孩子$v$，这已经走了一步，然后还有走回$u$所以再乘一个$f_v$，然后再乘一个$f_u$指走到父亲节点），化一下就是$d\cdot f_u =x+xf_u \left(\sum_{v\in son(u)} f_v \right)$。

（如果不能理解可以回顾一下求步数的$dp$，$f_u=\frac{1}{d}(1+\sum (1+f_u+f_v))$，然后加法改成乘法）

求导得到$d\cdot f’u=1+f_u \left(\sum\limits{v} f_v\right)+xf’u \left(\sum\limits{v}f_v \right)+xf_u \left(\sum\limits_{v} f’_v \right)$。

当$x=1$的时候$f_u(x)=1$，可以化简为$d\cdot f’_u=1+(d-1)+f’u (d-1)+\left(\sum\limits{v} f’_v \right)$，也就是$f’u=d+\left(\sum\limits{v} f’_v \right)$

两条路径乘起来求二阶导只需要知道一阶导和二阶导。

$(abc)’’=a’’bc+ab’’c+abc’’+2a’b’c+2a’bc’+2ab’c’$

继续求两阶导得$d\cdot f’’u=\sum\limits{v} (x f’’_u f_v+xf_uf’’_v +2f’_u f_v+2f_u f’_v+2xf’_u f’_v)$。

化简可得$d\cdot f’’u =\sum\limits{v} (f’’_u +f’’_v +2f’_u +2f’_v+2f’_u f’_v)$。

然后移项$f’’_u =\sum\limits_v f’’_v +2f’_u +2f’_v+2f’_u f’_v$，把右边的东西加起来即可。

然后这样就求出了每条边的二阶导。

然后维护一阶导二阶导，答案就是把每条边的生成函数乘起来。

$(ab)’=a’b+ab’,(ab)’’=a’’b+ab’’+2a’b’$

现在要求两个子树的答案。

考虑维护从下往上到这个点常数项一阶导二阶导的和，和从这个点到子树一些路径乘起来的一阶导和二阶导，和一次导的和，然后再乘上中间这条路径，同样的方法维护一下。用倍增合并一下路径上的信息即可。
